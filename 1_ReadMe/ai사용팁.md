# 💡 Antigravity 에이전트 쿼터 방어 및 효율화 팁

Antigravity IDE에서 에이전트 주도(Agent-driven) 개발 시 쿼터 낭비를 막고 효율을 극대화하는 4가지 핵심 전략입니다.

---

## 1. 🧠 모델 라우팅 전략: "기획은 가볍게, 실행은 무겁게"
Antigravity는 여러 모델(Gemini 3.1 Pro, Flash, Claude 등)을 선택할 수 있습니다. 무조건 가장 똑똑하고 무거운 모델만 쓰면 쿼터가 순식간에 증발합니다.

* **가벼운 모델 (예: Gemini 3.1 Flash):** 단순한 코드 자동완성, 주석 작성, 정규식 작성, 간단한 버그 찾기, 초기 폴더/파일 구조 잡기(Scaffolding)에 사용하세요. 빠르고 쿼터 소모가 거의 없습니다.
* **무거운 모델 (예: Gemini 3.1 Pro / Claude 등):** 전체 아키텍처 설계, 복잡한 비즈니스 로직 구현, 여러 파일이 얽힌 난해한 버그 수정 등 '진짜 머리를 써야 하는' 작업에만 아껴서 투입하세요.

## 2. 🗂️ 컨텍스트 다이어트: "필요한 파일만 먹여주세요"
에이전트가 코드를 분석할 때(특히 탭이나 에디터에 열려있는 파일들), 눈에 보이지 않게 엄청난 양의 토큰을 읽어들입니다.

* **불필요한 파일 닫기:** 질문할 때 참조하지 않아도 되는 파일은 에디터에서 닫아두거나, 에이전트의 컨텍스트 참조 범위에서 명시적으로 제외하세요.
* **`.antigravityignore` 활용:** 로그 파일, 빌드 결과물, 거대한 더미 데이터 JSON 등은 에이전트가 절대 읽지 못하게 ignore 처리해야 헛된 쿼터 낭비를 막을 수 있습니다.

## 3. ✂️ 태스크 쪼개기 (Micro-Management): "한 번에 하나씩 지시하기"
"게시판 CRUD 기능 전체를 만들어줘"라고 지시하면 에이전트가 수많은 파일을 오가며 자율적으로 코딩(Agent-driven)을 시도합니다. 이때 중간에 방향이 어긋나면 그동안 쓴 쿼터는 모두 날아갑니다.

* **에이전트 보조(Agent-assisted) 방식으로 전환:** "1단계: DB 스키마 작성해줘", "2단계: API 라우터 세팅해줘" 식으로 잘게 쪼개서 지시하세요. 
* 결과물을 확인하고 승인(Accept)한 뒤 다음 단계로 넘어가는 것이 한 번에 큰 작업을 맡기는 것보다 쿼터 생존율이 훨씬 높습니다.

## 4. 📜 명확한 룰(Rules) 설정: "재작업(Retries) 방지"
에이전트가 코드를 짰는데 우리 팀의 컨벤션(예: "상태 관리에 Redux 대신 Zustand만 쓴다", "변수명은 무조건 카멜케이스")과 달라서 다시 짜달라고 요청하면 쿼터가 두 배로 듭니다.

* 프로젝트 루트에 팀의 코딩 스탠다드와 프롬프트 지시사항을 담은 기본 규칙 파일(예: `.antigravityrules` 등)을 만들어 두세요. 
* 에이전트가 처음부터 우리 팀의 입맛에 맞게 코드를 생성하므로 불필요한 재수정 요청을 대폭 줄일 수 있습니다.
